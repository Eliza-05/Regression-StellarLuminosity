ISSUES & MISSING ELEMENTS
------------------------------------------------------------
- [Bug, Notebook 1] MSE formula precedence error:
    `mse = (1/2*n) * np.sum(error ** 2)`
  evaluates as `(n/2) * sum(error^2)` in Python.
  Correct code: `mse = (1/(2*n)) * np.sum(error ** 2)`
  Gradient functions are not affected (they do not call error_mse internally),
  so gradient descent converges correctly, but all printed/plotted cost values
  are wrong by a factor of n^2.

- [Bug, Notebook 1] Non-vectorized section uses `M_list` (lowercase 'l') in
  the loop inside `predict_noVec` but the parameter is named `M_List`
  (capital 'L'). This would raise a NameError at runtime.

- [Compliance] `pandas` is pip-installed in both notebooks and imported in
  Notebook 2. Pandas is not in the allowed library list. Although it is not
  used in any regression computations, its presence violates the stated
  constraint. The pip install cell should be limited to numpy and matplotlib.

- [Minor, Notebook 1] The convergence plot starts at iteration index 10
  (`history[10:]`) to skip the initial steep drop, but this detail is not
  explained. A brief comment would improve clarity.

- [Minor, Notebook 2] Convergence plots for M1/M2/M3 are generated inside the
  training loop rather than as a comparative multi-curve figure. A side-by-side
  or overlaid convergence comparison would make the M1/M2/M3 differences more
  visually immediate.

TA FEEDBACK TO STUDENT
------------------------------------------------------------
Overall, this is a well-executed submission that clearly demonstrates mastery
of the core concepts: gradient descent, feature engineering, convergence
analysis, and cloud deployment. The structure is clean and the mandatory
components are all present and well-explained.

The main issue to address is the MSE implementation bug in Notebook 1. Always
verify formula implementations against hand-computed examples: for n=10 data
points and residuals of ~1, the MSE should be in the range of ~0.05–1.0, not
~50–100. A quick sanity check would have caught this. The fix is a single
parenthesis: change `(1/2*n)` to `(1/(2*n))`.

The variable name mismatch (`M_list` vs `M_List`) in the non-vectorized
section is a small but critical bug — Python is case-sensitive. It is good
practice to test each function independently with a small example before
embedding it in a longer pipeline.

Regarding pandas: the assignment explicitly restricts libraries to Python,
NumPy, and Matplotlib. Even if pandas is not used in computations, importing
it signals non-compliance. Remove the pandas import and restrict pip installs
to the two required libraries.

The SageMaker evidence is excellent — detailed, well-documented, and supported
by multiple screenshots. The local-vs-cloud comparison adds real value to the
README.

For future work, consider plotting the M1/M2/M3 convergence curves on the same
axes so that the improvement from each additional feature term is immediately
visible.
